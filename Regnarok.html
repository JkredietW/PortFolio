Robinson's Island

School project 3-1. (9 weeks)
Team: Jorn(programmer), Fady(character artist), Timme(programmer).

This project was mostly about learning new things. We wanted to just improve our skills. This is the reason why this team exists just of 3, instead of the normal 4-5 size.

What we did

I focused mainly on the character controller, stats calculations, UI-functionality and photon(multiplayer).
Timme focused on the map generation and AI for the hostile creatures.
For fady it wasn't really a challange, so we just asked for a few models. With this it wouldn't consume too much of his time and he could further improve his own skills.

This game is a survival game, we used Muck as a reference. You start on an island and will need to collect a few items to craft weapons/tools.

Around the map you'll see some chests and totems. These totems can spawn enemies, slain enemies drop a few gold coins. These gold coins are used to open chests that contain items. â€‹These items are different from the normal resources. These items a pure for stats and can stack infinitely. You will need stackable items to do enough damage for the bosses. 

The bosses can be spawned with the special boss totems. After doing all 4 boss totems the big boss will spawn, once beaten you will have completed the game.

Photon - Multiplayer

In short:

We made sure to do all the networking on the host-client. Once someone changes something the client will notify the host and the host will apply the changes on all clients.

Not so short:

This project we wanted to add multiplayer. Mostly because i wanted to understand multiplayer coding a bit more. It also connected nicely with the survival genre. 
 Multiplayer already starts when the game is launched. It will conect to the Photon server(eu). Once connected the menu will appear. 1 person can create a host lobby, the others need to connect through the join lobby. This lobby will show all availible rooms to join.

When the hosts starts the game, the host-client will send the seed used to all other clients. With this seed the island is generated(done by timme).
After the map has been generated a nav-mesh map is also generated. This is done only on the host. This will be done only on the host, because the hostile creatures only need to behave on the host-client. The host client then sends the information of the enemies to the other clients. For the generated objects such as the trees, stones and bamboo. They will be generated on each client using the earlier send seed. When attacking a tree, the tree will notify the host-cient with what it has been hit. This will then be send to all other clients. With this all damage is synced. This also happens for the items, enemies and stackable items.
