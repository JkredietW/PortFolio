Astar

To be honest, i just wanted to move objects around without relying on the default Unity pathfinding. This is because the default unity pathfinding can't generate the pathfinding during playtime.

With my own Astar pathfinding i can choose where i want enemies to go and generate a new grid if needed while in playmode. With this i can add/remove objects on the grid in playtime. The objects that use this grid will also know that this happened and will react according to the changes.

How Astar works

A* is one of the many pathfinding methods. It tries to find a path towards the destination with using the least possible tiles. Heres how it works. At the moment it starts to search for a path every tile get a few numbers: Gcost, Hcost and Fcost.

Gcost is how far away the tile is from the start tile.
Hcost is how far away the tile is from the end tile.
Fcost is the total cost combining G- and H-cost.

Now that every tile has these numbers it will start checking neiboring tiles to see wich of them has the lowest Hcost, if it is the same it will start the check the Fcost. This will go on with every tiles till it reaches its destination.

The grid also has these yellow tiles, those tiles are non-walkable tiles. When the grid gets generated every tile checks if it is colliding with an object with the layermask wall. If it is it will set a boolean to false and the pathfinding will ignore this tile.
