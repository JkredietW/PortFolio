State machine

School project 3-1. (9 weeks)
I was making a small colony-sim, because it looked kinda amusing to make. Having all those tiny people walk around. 

After i had a few functions for walking/eating/sleeping, but i was getting a bit out of hand with all the if-statements. Thats when i remembered a friend that was doing something similar mentioning State machine.

After doing some research about it. I came to the conclusion that it would be a nice selution. So i implemented it and yes, it nice to just swap from behaviors when needed by calling one function. It also became real nice to just add another behavior to my little humans.

This video shows how the tiny humans behave.

I made 4 roles: Workless, woodcutter, miner and farmer.
Workless humans walk around and do nothing. Like the one being selected in the video. This particular one has a lot of hunger and gets fast hungry. When enter the hungry-state humans will check if the house has any food stored and go there to eat. If the house has nothing they will search for the nearest foodsource. Foodsources are the green balls.

Woodcutters, miners and farmers will search for their job specific source to gather resources. When they cant carry any more they will go to the house(blue box) and store what they are carrying. That being said, they will stop working if they are working for to long. If that happens they shall enter idle-state to relieve their stress.

This little project also has a day/night cycle. When it becomes night all humans will gather at the house(blue box) to sleep till morning. The player can also select a human to observe them and see what they are doing. If a human is selected you can move them around. This uses the Astar pathing.

The pathfinding used here is a more refined Astar than the one showed on my other page. This one only searches once for a path, instead of every frame.

How a state machine works

As the state in State machine says, you have state in wich the stateMachine cycles.

To put it simple, you swap between behaviors. You will only have to call the function that changes the state with the desired state. So instead of using if statements to decide what to do, you only give it one option and change that option every time you chance states.
